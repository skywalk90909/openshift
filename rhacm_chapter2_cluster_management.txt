The Core Idea: What is RHACM?
You already manage one OpenShift cluster daily. Now imagine your company has 20 OpenShift clusters — 5 on AWS, 3 on Azure, 4 on-prem in Hyderabad DC, 3 in Mumbai DR, and 5 edge clusters. RHACM is the single pane of glass that lets you manage all 20 from one console — create, import, upgrade, apply policies, check compliance.

text
WITHOUT RHACM:                    WITH RHACM:
  Login to cluster-1               One RHACM Hub Console
  Login to cluster-2         ──▶   See ALL 20 clusters
  Login to cluster-3               Manage from one place
  ... (20 times)
Two roles in RHACM:

Hub Cluster — the master OpenShift cluster where RHACM is installed. This is your control plane.

Managed Cluster — every other cluster that RHACM controls. They report status back to the hub.

2.5 & 2.6 — Configuring Access Control for Multicluster Management
Problem: You have 3 teams:

Platform team — should manage ALL clusters

App team A — should only manage dev and staging clusters

App team B — should only manage prod clusters

RHACM uses ManagedClusterSets to group clusters and RBAC to control access:

text
ManagedClusterSet: "non-prod"
  ├── dev-cluster
  └── staging-cluster
        │
        │ Bind to namespace: team-a-ns
        ▼
  RBAC: Team A users → admin access on "non-prod" ClusterSet
        Team A users → CANNOT see or touch prod-cluster

ManagedClusterSet: "production"
  ├── prod-aws-cluster
  └── prod-azure-cluster
        │
        │ Bind to namespace: team-b-ns
        ▼
  RBAC: Team B users → admin access on "production" ClusterSet
        Team B users → CANNOT touch dev/staging
The 3 objects that work together:

text
ManagedClusterSet      → groups clusters together
       +
ManagedClusterSetBinding → links ClusterSet to a namespace
       +
RBAC RoleBinding       → grants users access to that namespace
       =
Users can only see and manage their assigned clusters
bash
# Create ClusterSet
oc create managedclusterset production

# Add prod cluster to the set
oc label managedcluster prod-aws-cluster \
  cluster.open-cluster-management.io/clusterset=production

# Bind to namespace
oc create managedclustersetbinding production \
  --namespace=team-b-ns

# Grant Team B admin on that namespace
oc adm policy add-role-to-group admin team-b \
  --namespace=team-b-ns
2.7 & 2.8 — Troubleshooting Common Import Issues
Scenario: You try to import an existing on-prem OpenShift cluster into RHACM but it stays in Pending state forever.

Two import methods — and where each breaks:

text
Method 1: Auto Import (RHACM generates YAML → apply on managed cluster)
  ├── RHACM generates → import secret + klusterlet YAML
  ├── You run: oc apply -f import.yaml  (on managed cluster)
  └── klusterlet agent installs → phones home to hub

Method 2: kubeconfig Import (paste managed cluster kubeconfig in RHACM UI)
  ├── RHACM uses kubeconfig to directly connect to managed cluster API
  └── RHACM installs klusterlet agent automatically
Common failure flow and how to debug:

text
Symptom: Cluster stuck in "Pending" for >10 minutes
     │
     ▼
Check 1: Can hub reach managed cluster API?
  oc get managedcluster <cluster-name> -o yaml
  └── Look for: "ManagedClusterConditionAvailable: False"
  └── Reason: "Connection refused" → firewall blocking port 6443

Check 2: Is klusterlet running on managed cluster?
  oc get pods -n open-cluster-management-agent
  └── klusterlet pod CrashLoopBackOff?
      → Check logs: oc logs klusterlet-xxx -n open-cluster-management-agent
      → Common cause: Hub API URL not reachable from managed cluster

Check 3: Certificate/token expired?
  oc get secret -n open-cluster-management-agent
  └── bootstrap-hub-kubeconfig → check expiry
      → Re-generate import YAML from RHACM UI
Most common root causes:

Symptom	Root Cause	Fix
Stuck in Pending	Firewall blocks hub ↔ managed on port 6443	Open firewall rules both directions
Registration Failed	Bootstrap token expired	Re-import from RHACM UI (new token)
Available: Unknown	klusterlet pod down on managed cluster	Restart klusterlet deployment
Version mismatch	RHACM version too old for managed OCP version	Upgrade RHACM hub first
